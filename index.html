números

Números do projeto

Automação completa com timing gaussiano (que simula variação humana real), retomada parcial e tratamento robusto de todos os cenários de erro.

Camada 1 — Timestamp
Cada timestamp enviado tem variação aleatória que imita o atraso natural de rede humano, com drift por request único.
Camada 2 — Intervalo
O tempo entre requests segue uma curva gaussiana — não é fixo, como um humano que às vezes pausa, olha outra coisa, volta.
Camada 3 — Step
Distribuição bi-modal: 80% normal (5-9s) + 20% pausa curta (1-4s). Humanos não assistem vídeos em velocidade constante.

como usar

Guia completo — 6 passos

Qualquer pessoa consegue usar. Não precisa instalar nada — só o Discord aberto.

01
Habilite o DevTools (se necessário)
O Discord Stable bloqueia o console por padrão. Se F12 não abrir, habilite manualmente:
Windows
  • Abra: %appdata%/discord/settings.json
  • Adicione a linha abaixo e reinicie:
  • "DANGEROUS_ENABLE_DEVTOOLS_ONLY_ENABLE_IF_YOU_KNOW_WHAT_YOURE_DOING": true
Linux / macOS
  • Abra: ~/.config/discord/settings.json
  • Adicione a mesma linha e reinicie
Alternativa
Use Discord PTB ou Canary — têm DevTools habilitado por padrão. Funciona igual ao Stable para missões.
02
Inscreva-se nas missões
Abra o Discord → aba Missões → inscreva-se manualmente. Missões de jogos exigem o jogo instalado — o script não consegue contornar isso.
03
Abra o console do Discord
Use o atalho do seu sistema:
Windows / Linux
  • F12 direto
  • Ctrl+Shift+I
macOS
  • Cmd+Opt+I
  • F12 não funciona no Mac
Dica
Se aparecer "allow pasting", digite isso no console e pressione Enter antes de colar.
04
Clique na aba Console
No DevTools que abriu, clique em Console no topo da janela.
05
Cole e pressione Enter
Clique em Copiar Script acima, cole com Ctrl+V e pressione Enter. O script verifica o servidor, lista as missões e processa automaticamente. Para parar a qualquer momento: yKauttizStop()
06
Colete as recompensas
Após o script terminar, abra a aba Missões no Discord e colete recompensas e orbs manualmente. A coleta automática não funciona por design da plataforma.

Perguntas frequentes

Não. O script é JavaScript puro que roda dentro do console do seu próprio Discord. Ele não baixa nada, não instala nada, não acessa arquivos do computador e não envia senhas ou dados pessoais a lugar nenhum. Quando você fecha o Discord, o script para e some completamente. O código aparece protegido para preservar a autoria de yKauttiz, não para esconder algo malicioso.
O script é distribuído protegido para preservar a autoria de yKauttiz e evitar redistribuição não autorizada. É uma prática comum entre desenvolvedores para proteger seu trabalho. O Discord executa normalmente e os logs aparecem coloridos como no preview acima.
O servidor é o canal oficial de suporte, atualizações e distribuição. O script verifica sua presença e tenta entrar automaticamente ao iniciar. Se por algum motivo não conseguir, entre manualmente em discord.gg/xv6K5nVezS e rode novamente. Sem verificação do servidor, o script não executa.
USE POR SUA CONTA E RISCO. O script usa APIs internas do Discord de forma não oficial, o que pode violar os Termos de Serviço. yKauttiz não se responsabiliza por banimentos, suspensões ou qualquer consequência. Se quiser mais segurança, use em uma conta secundária.
Missões de jogos específicos exigem que o jogo esteja instalado e vinculado à sua conta Discord. Essa validação é feita pelo próprio Discord de forma que não pode ser contornada via script. Inscreva-se manualmente na aba Missões antes de rodar.
O Discord processa recompensas internamente com validação anti-fraude. A coleta automática não funciona por design da plataforma. Após o script terminar, basta abrir a aba Missões no Discord — as recompensas e orbs estão lá esperando.
WATCH_VIDEO e PLAY_ACTIVITY funcionam no navegador e no Desktop App.

PLAY_ON_DESKTOP e STREAM_ON_DESKTOP funcionam apenas no Discord Desktop App. No navegador, esses tipos são detectados e pulados automaticamente com aviso no console.
Alguns builds do Discord exigem confirmação antes de permitir colar código. Quando aparecer, digite allow pasting no console e pressione Enter. Depois cole o script normalmente. É uma proteção do Discord e não afeta o funcionamento.
O Discord Stable bloqueia o DevTools por padrão. Para habilitar:

1. Feche o Discord
2. Abra %appdata%/discord/settings.json (Windows) ou ~/.config/discord/settings.json (Linux/Mac) em um editor de texto
3. Adicione: "DANGEROUS_ENABLE_DEVTOOLS_ONLY_ENABLE_IF_YOU_KNOW_WHAT_YOURE_DOING": true
4. Salve e reinicie o Discord

Alternativa: use o Discord PTB ou Canary — têm DevTools habilitados por padrão e funcionam igual ao Stable para missões.
Digite yKauttizStop() no console e pressione Enter. O script finaliza o step atual, restaura todos os stores modificados (GameStore, StreamStore) e exibe o resumo parcial da sessão. Os dados de sessão ficam disponíveis via window.yKauttizLog.
